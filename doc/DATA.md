# Данные и назначение репозитория

Документ описывает, **какие данные** использует учебный RAG, **откуда** они взяты и **что именно** попадает в индекс.

---

## Назначение репозитория

**Контекст продукта:** прототип контентного ядра для **«Умняша»** — AI-помощника по домашним заданиям для младшей школы. Ключевая идея: помощник отвечает **не из «головы» модели, а строго по проверенным методическим материалам** (учебникам и книгам для учителя). RAG здесь не учебная абстракция, а архитектурное требование: в детском edtech важны точность и соответствие методическим нормам, поэтому ответы должны опираться на конкретный источник и показывать его.

**Что демонстрирует репозиторий:**

- полный offline-pipeline: PDF учебника → документы → чанки → TF-IDF индекс → поиск → demo-ответ;
- ответ **только по найденным фрагментам** с указанием источника (`doc_id`, score);
- явный **отказ**, если релевантного контекста нет (вопрос не по учебнику);
- Streamlit UI для интерактивной проверки.

**Границы MVP:**

- поиск по **словам** (TF-IDF), не embeddings и не LLM;
- demo-режим без внешних API;
- локальный корпус из одного учебника (154 документа, ~1317 чанков);
- не production-система, а **учебный шаблон** RAG-конвейера.

## Источник данных

| Источник | Файл в проекте | Комментарий |
|----------|----------------|-------------|
| УМК «Английский в фокусе» (Spotlight 2), книга для учителя | `data/raw/pdf/Spotlight_2_TB.pdf` | PDF с текстовым слоем. **Не коммитится** — защищён авторским правом (см. `.gitignore`). |
| Подготовленный корпус | `data/raw/datasets.json` | 154 записи: `id`, `name`, `text`. Производный учебный текст, извлечённый из PDF. |
| Скрипт подготовки | `scripts/pdf_to_datasets.py` | Извлечение текста из PDF, сегментация по урокам/модулям, очистка артефактов. |

**Что это за учебник:** «Английский в фокусе» (Spotlight) для 2 класса, книга для учителя (Teacher's Book) — методические материалы: планы уроков, активная лексика, фонетика, сценарии занятий по пяти модулям (My Home!, My Birthday!, My Animals!, My Toys!, My Holidays!).

**Авторские права:** учебник защищён авторским правом (Express Publishing / «Просвещение»). В репозиторий **не коммитится сам PDF**; хранится только производный обработанный текст для локальной демонстрации pipeline. Для публичного распространения полнотекстовых учебников требуется лицензия.

**Дата подготовки:** июнь 2026.

## Что индексируем

| Поле / артефакт | Индексируется? | Где используется |
|-----------------|:--------------:|------------------|
| `text` из `datasets.json` | **Да** | TF-IDF матрица, поиск, demo-ответ |
| `name` | Нет (метаданные) | UI, источники — название урока/модуля |
| `doc_id` | Нет (метаданные) | UI, источники — идентификатор документа |
| `source_file` | Нет | `documents.jsonl`, трассировка происхождения |

**Pipeline:**

```
Spotlight_2_TB.pdf → datasets.json → documents.jsonl → chunks.jsonl → vectorizer.pkl + matrix.npz
```

- **Сегментация:** по урокам (маркер «Начало урока») и модулям; длинные блоки дробятся на под-документы до 3000 символов (`scripts/pdf_to_datasets.py`).
- **Чанки:** нарезка по абзацам, max 400 символов, overlap 50 (`app/chunker.py`).
- **Поиск:** cosine similarity по TF-IDF векторам (`app/retriever.py`).

**Масштаб корпуса:** 154 документа → **~1317 чанков** после нарезки.

---

## Что не индексируем

| Не индексируется | Причина |
|------------------|---------|
| `data/raw/pdf/*.pdf` | Исходный учебник — только для локальной подготовки `datasets.json`; не коммитится |
| Оглавление, выходные данные, «PHOTOCOPIABLE MATERIAL» | Служебные страницы, отфильтрованы при очистке |
| Секреты, API-ключи | Demo-режим без внешних LLM |
| `data/processed/*.jsonl` | Промежуточные артефакты, генерируются скриптами |
| `data/index/*` | Индекс пересобирается командой `build_index.py` |

## Demo-проверка

**Вопросы с ответом (есть в учебнике):**

1. «Что такое УМК Английский в фокусе?»
2. «Как работать с диалогом на уроке?»
3. «Как используются картинки на уроке?»

**Negative-вопрос (нет в учебнике) → отказ:**

- «Столица Австралии» — ни одно слово не встречается в корпусе, система корректно отказывает.

## Как обновить данные

1. Положить PDF учебника в `data/raw/pdf/` (локально, не коммитится).
2. Запустить извлечение: `uv run python scripts/pdf_to_datasets.py data/raw/pdf/Spotlight_2_TB.pdf`
3. Пересобрать индекс: `uv run python scripts/build_index.py`

Скрипт принимает несколько PDF за один прогон — можно собрать общий корпус из нескольких учебников.
