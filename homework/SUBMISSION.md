## Что сделано

Учебный RAG на базе шаблона репозитория, адаптированный под собственный проект — **«Умняш»** (AI-помощник по домашним заданиям для младшей школы). Идея: помощник отвечает строго по методическим материалам учебника, а не из «головы» модели.

**Данные:** книга для учителя УМК «Английский в фокусе» (Spotlight 2). Из PDF извлечён текст, сегментирован по урокам/модулям, очищен от артефактов извлечения. Подробно — в [`doc/DATA.md`](../doc/DATA.md).

- 154 документа в `datasets.json`
- ~1317 чанков после нарезки

**Pipeline:** PDF → datasets.json → documents.jsonl → chunks.jsonl → TF-IDF индекс → retrieval → demo-ответ → Streamlit UI. Полностью воспроизводим (см. README).

## Реализованное улучшение

Относительный критерий отказа (`top1 + gap` вместо абсолютного порога) — см. [`IMPROVEMENTS.md`](../IMPROVEMENTS.md). Решает проблему: на длинном методическом тексте абсолютный порог не разделяет валидные вопросы и шум, потому что частые слова дают ненулевой score почти на любой запрос.

## Запуск

```bash
uv sync
uv run python scripts/pdf_to_datasets.py data/raw/pdf/Spotlight_2_TB.pdf
uv run python scripts/build_index.py
uv run streamlit run app/main.py
```

> PDF учебника не включён в репозиторий (авторские права); в репозитории — производный обработанный `datasets.json`.
